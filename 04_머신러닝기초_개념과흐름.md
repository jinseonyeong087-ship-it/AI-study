# 4. 머신러닝 기초: 개념과 흐름 (완전 입문자 버전)

> 목표: "머신러닝이 뭔지" + "불량검출에서 어떻게 쓰는지"를 감으로 이해하기

---

## 1) 머신러닝이 뭐야?

아주 쉽게 말하면:
- **일일이 규칙을 사람이 코딩하지 않고**
- **데이터를 보고 규칙을 스스로 찾게 하는 방법**

예시)
- 전통 방식: "스크래치 길이가 3mm 이상이면 불량" 같은 규칙을 사람이 직접 작성
- 머신러닝: 정상/불량 예시를 많이 보여주면 모델이 경계(규칙)를 학습

---

## 2) 불량검출에서 왜 중요해?

현업에서는 불량 종류가 계속 바뀌고, 조명/각도/재질도 바뀝니다.
규칙 기반만으로는 한계가 빨리 와요.

머신러닝/딥러닝은:
- 조건이 조금 달라져도 대응 가능
- 데이터가 늘수록 성능 개선 가능
- 반복 검사 자동화에 유리

---

## 3) 꼭 알아야 할 단어 (진짜 핵심만)

- **데이터(Data)**: 학습에 쓰는 재료
- **라벨(Label)**: 정답 (정상/불량)
- **특징(Feature)**: 모델이 판단에 쓰는 정보
- **모델(Model)**: 데이터를 학습한 "판단기"
- **학습(Train)**: 모델이 규칙을 찾는 과정
- **추론(Inference/Predict)**: 학습된 모델로 실제 데이터 판단

---

## 4) 머신러닝의 기본 흐름 (현업 공통)

1. **문제 정의**
   - 우리는 뭘 맞추고 싶은가?
   - 예: "이미지 1장 단위로 정상/불량 분류"

2. **데이터 수집**
   - 정상/불량 이미지 확보
   - 가능한 실제 라인 조건 반영

3. **데이터 정리(전처리)**
   - 파일 깨짐, 중복 제거
   - 라벨 확인(오라벨 수정)

4. **데이터 분리**
   - 학습(train) / 검증(val) / 테스트(test)

5. **모델 학습**
   - 알고리즘 선택 후 학습

6. **성능 평가**
   - 정확도만 보지 말고 정밀도/재현율 확인

7. **개선 반복**
   - 실패 케이스 모으기 → 데이터 보완 → 재학습

---

## 5) 정확도만 보면 위험한 이유

불량검출에서 자주 생기는 상황:
- 정상 950장, 불량 50장 (불균형)
- 모델이 전부 정상이라고 예측해도 정확도 95%

근데 불량을 못 잡으면 현업에서는 실패.
그래서 **재현율(불량을 놓치지 않는 비율)** 이 중요할 때가 많습니다.

---

## 6) 현업에서 자주 듣는 질문 (면접 아님, 실무형)

### Q1. "왜 오늘 성능이 떨어졌지?"
체크 순서:
1) 카메라/조명 바뀌었나?
2) 신규 불량 유형이 생겼나?
3) 라벨 품질 문제 있나?
4) 최근 데이터 분포가 달라졌나?

### Q2. "정확도 높은데 왜 현장 불만이 많지?"
- 실제로는 미검출(놓침)이 문제일 수 있음
- 지표를 정확도만 본 경우가 많음
- 불량 클래스별 재현율을 따로 봐야 함

### Q3. "모델을 바꿔야 하나, 데이터를 늘려야 하나?"
초기엔 대부분 **데이터/라벨 품질 개선**이 더 효과 큼.

---

## 7) 오늘의 미니 실습 (개념 확인용)

아래 질문에 답해보면 이해가 빨라집니다.

1. 우리 문제는 분류/탐지/분할 중 무엇인가?
2. 불량을 놓치면 어떤 비용이 발생하는가?
3. 그래서 우선 지표는 정확도/재현율 중 무엇인가?
4. 현재 데이터에서 가장 불안한 부분은 무엇인가? (조명? 라벨? 수량?)

이 4개만 명확해도, 현업 대화 이해도가 확 올라갑니다.
