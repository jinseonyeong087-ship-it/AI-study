## 1. 서버 개념 정리

### 서버란 무엇인가? (역할 중심)
서버는 한 줄로 말하면 **요청을 받아 처리하고 결과를 돌려주는 프로그램(또는 그 프로그램이 실행되는 컴퓨터)** 입니다.

- 클라이언트(웹/앱)가 요청을 보냄
- 서버가 처리 로직 실행
- DB 조회/저장, AI 호출 등 수행
- 응답 반환(JSON, 이미지, 파일 등)

**한 줄 핵심:** 서버는 “정보를 저장하는 상자”가 아니라 **요청 처리 엔진**이다.

### 로컬 서버 vs 실제 서버(IDC, AWS)

| 구분 | 로컬 서버 | 실제 서버(IDC/AWS) |
|---|---|---|
| 위치 | 내 PC | 데이터센터/클라우드 |
| 목적 | 개발/테스트 | 운영(실서비스) |
| 접근 | 보통 내 PC만 | 외부/사내망 사용자 접근 |
| 안정성 | 낮음(PC 꺼지면 종료) | 높음(24시간 운영) |

### Express / FastAPI가 서버인 이유
- Express(Node.js), FastAPI(Python)는 **HTTP 요청을 받아 라우팅하고 처리하는 프레임워크**
- 즉, 단순 라이브러리가 아니라 **서버 애플리케이션의 뼈대**

## 2. 백엔드 역할

### 프론트 ↔ 백엔드 ↔ DB ↔ AI 전체 흐름

```text
[프론트(웹/앱)]
    ↓ HTTP 요청
[백엔드 API 서버]
    ├─ DB 조회/저장
    ├─ AI 모델 추론 호출
    └─ 비즈니스 로직 처리
    ↓
[JSON 응답]
```

### 백엔드는 “연결”이 아니라 “처리”
많이 오해하는 부분: 백엔드는 단순 중계기가 아닙니다.

실제로 백엔드는 다음을 담당:
- 입력 검증 (필수값, 타입, 범위)
- 권한 확인 (로그인/JWT/역할)
- 비즈니스 규칙 적용 (예: 재검사 조건)
- 트랜잭션 처리 (DB 일관성)
- 로그/감사 추적

**한 줄 핵심:** 백엔드는 시스템의 “교통경찰”이자 “업무 엔진”이다.

### 실제 코드 흐름 예시 (개념)
```python
# 1) 요청 수신
POST /api/inspect

# 2) 입력 검증
if image is None:
    return 400

# 3) 권한 확인
verify_jwt(token)

# 4) AI 추론
result = model.predict(image)

# 5) DB 저장
save_inspection(result)

# 6) 응답
return {"status": "ok", "result": result}
```

## 3. 웹서버 / WAS / DispatcherServlet 개념

### Nginx 역할 (웹서버/리버스 프록시)
Nginx는 보통 앞단에서:
- HTTPS 종료(인증서)
- 정적 파일 제공
- 요청을 내부 앱 서버로 전달(Reverse Proxy)
- 로드밸런싱

### Spring DispatcherServlet 역할
Spring MVC에서 DispatcherServlet은:
- 모든 요청을 먼저 받는 **Front Controller**
- URL에 맞는 Controller로 전달
- 응답 생성 과정을 조정

### FastAPI에서 대응 구조
FastAPI도 구조적으로 유사:
- `app = FastAPI()`가 앱 엔트리
- 라우터가 URL 분기
- 함수(엔드포인트)가 Controller 역할 수행
- Uvicorn이 ASGI 서버로 요청 실행

**한 줄 핵심:** 기술은 달라도 “요청 수신 → 라우팅 → 처리 → 응답” 구조는 같다.

## 4. FastAPI 구조

### `app = FastAPI()` 의미
- API 앱의 시작점(애플리케이션 객체)
- 라우터, 미들웨어, 예외처리 등록 대상

### 라우터 개념
- URL 묶음을 기능 단위로 분리 (`/auth`, `/products`, `/ai`)
- 코드 유지보수성을 높임

### API 정의 구조 예시
```python
from fastapi import FastAPI, APIRouter

app = FastAPI()
router = APIRouter(prefix="/api/ai", tags=["AI"])

@router.post("/detect")
async def detect(file: bytes):
    return {"ok": True}

app.include_router(router)
```

### Uvicorn 역할
- FastAPI 코드를 실제 HTTP 서버로 실행하는 런타임
- 예: `uvicorn app.main:app --reload`

## 5. REST API 개념

### REST API란?
REST는 HTTP 기반 리소스 설계 원칙입니다.
- URL은 자원 중심 (`/inspections`, `/cameras`)
- 동작은 HTTP 메서드로 구분 (`GET`, `POST`, `PUT`, `DELETE`)

### Controller와의 차이
- Controller: 코드 구조/역할 이름
- REST API: URL/메서드/상태코드 설계 방식

### URL 설계 예시
| 목적 | 메서드 | URL |
|---|---|---|
| 검사 목록 조회 | GET | `/api/inspections` |
| 검사 1건 조회 | GET | `/api/inspections/{id}` |
| 검사 생성 | POST | `/api/inspections` |
| 검사 수정 | PUT | `/api/inspections/{id}` |
| 검사 삭제 | DELETE | `/api/inspections/{id}` |

**한 줄 핵심:** REST는 “예쁘게 짓는 규칙”이 아니라 **협업 가능한 API 약속**이다.

## 6. AI 개념 정리

### 머신러닝 vs 딥러닝
- 머신러닝: 사람이 특징(Feature)을 설계하는 비중 큼
- 딥러닝: 신경망이 특징을 자동 학습

### Classification / Detection / Segmentation
| 작업 | 출력 | 사용 상황 |
|---|---|---|
| Classification | 이미지 전체 클래스 1개(또는 다중) | 제품 종류 분류 |
| Detection | 객체 박스 + 클래스 | 불량 위치 대략 확인 |
| Segmentation | 픽셀 단위 마스크 | 불량 면적/경계 정밀 분석 |

### YOLO란?
YOLO는 대표적인 실시간 객체탐지 모델 계열.
- 빠름
- 산업 현장 PoC에 많이 쓰임
- 디텍션/세그 모델 라인업 존재

### 불량검출에서의 사용
- 초기 도입: YOLO 디텍션으로 빠른 검증
- 고도화: 세그멘테이션으로 미세 불량 경계 분석

## 7. AI + 서버 연결 구조

### 이미지 → 모델 → 결과 흐름
```text
클라이언트 이미지 업로드
   ↓
FastAPI 엔드포인트 수신
   ↓
전처리(리사이즈/정규화)
   ↓
모델 추론(YOLO/분류기)
   ↓
후처리(NMS/임계치/맵핑)
   ↓
DB 저장 + JSON 응답
```

### FastAPI로 AI 서버 만드는 구조
- `routers/ai.py` : 요청 입구
- `services/ai_service.py` : 추론/후처리 핵심
- `models/*.py` : DB 엔티티
- `dependencies.py` : DB 세션 주입

### 코드 흐름 예시
```python
@router.post('/detect')
async def detect(file: UploadFile, db: Session = Depends(get_db)):
    image_bytes = await file.read()
    result = analyze_image(image_bytes, db)  # 모델 + DB 매핑
    return {"result": result}
```

**한 줄 핵심:** AI 성능만으로 서비스가 완성되지 않는다. 서버 연결/저장/응답 설계가 완성도를 만든다.

## 8. 보안 구조

### JWT vs API Key
| 항목 | JWT | API Key |
|---|---|---|
| 목적 | 사용자 인증/인가 | 서비스 간 호출 식별 |
| 정보 | 사용자/권한 클레임 포함 | 단순 키 문자열 |
| 만료 | 보통 있음 | 없거나 수동 로테이션 |

### IP 화이트리스트 / VPN
- IP 화이트리스트: 허용된 IP만 접근
- VPN: 사설망 안에서만 시스템 접근

### 권한 분리: 같은 API vs 분리 API
- 같은 API + 역할(Role) 분기: 운영은 간단
- API 자체 분리(내부/외부): 보안 경계 명확

### 실무 조합 예시
- 사용자 앱: JWT
- 공장 장비/게이트웨이: API Key + IP 제한
- 관리자 페이지: JWT + MFA + VPN

## 9. DB 설계

### 데이터 흐름 기반 설계 방법
테이블은 화면 기준이 아니라 **업무 이벤트 흐름** 기준으로 나눕니다.

### 예시: camera / inspection / detection
- `camera` : 카메라 메타정보(설치위치, 라인)
- `inspection` : 검사 1회 실행 단위(시작시간, 종료시간, 상태)
- `detection` : 검사 내 검출 객체들(클래스, confidence, bbox/mask)

```text
camera (1) ─── (N) inspection (1) ─── (N) detection
```

### 왜 분리하나?
- 조회 성능 (필요한 데이터만 빠르게)
- 변경 영향 최소화
- 이력/추적 용이
- 대량 데이터 관리에 유리

### 데이터 증가량 고려
- detection은 가장 빨리 증가
- 보통 파티셔닝/아카이빙 정책 필요
- 원본 이미지는 객체스토리지(S3 등)로 분리 저장 권장

## 10. 실무 전체 구조 정리

### 공장 불량검출 시스템 아키텍처

```text
[카메라/엣지장치]
   ↓ 이미지 전송
[API Gateway / Nginx]
   ↓
[FastAPI 백엔드]
   ├─ 인증/권한
   ├─ 검사 요청 관리
   ├─ AI 추론 서비스 호출
   └─ DB 기록
        ↓
     [RDB + 객체스토리지]
        ↓
[대시보드/리포트]
```

### 신입이 맡는 역할 (현실 버전)
1. API 스펙 이해 및 엔드포인트 구현 보조
2. 요청/응답 로깅, 예외 처리, 입력 검증
3. 라벨 데이터 정리 및 학습 파이프라인 실행 지원
4. 모델 결과 DB 저장/조회 API 연결
5. 장애 재현 및 원인 분류(데이터/모델/서버)

**한 줄 핵심:** 신입의 강점은 ‘새 모델 개발’보다 **흐름을 끊기지 않게 만드는 안정적 구현**이다.

## 부록 A. 초보가 자주 헷갈리는 질문

### Q1. FastAPI가 서버야, Uvicorn이 서버야?
- FastAPI는 애플리케이션 프레임워크
- Uvicorn은 그 앱을 실행하는 ASGI 서버

### Q2. AI 정확도만 높이면 끝?
- 아니요. 실제 서비스는 지연시간, 장애복구, 로그, 보안, 데이터 품질이 같이 중요

### Q3. 라벨링 툴은 무엇을 고르면 돼?
- 개인 빠른 시작: labelImg/labelme
- 팀 협업 운영: CVAT
- 웹 기반 반복 실험: Roboflow
