# 13. 디텍션/세그멘테이션 환경 세팅 + 실행 방법 (노베이스 실무 입문)

> 목표: "설치만 해둔 상태"가 아니라, **직접 열고 실행해서 결과 확인**까지 할 수 있게 정리

---

## 0) 먼저: 무엇을 세팅하는가?

디텍션/세그멘테이션을 하려면 보통 아래가 필요합니다.

1. Python 실행 환경 (가상환경)
2. 필수 패키지 (ultralytics, opencv 등)
3. 데이터셋 폴더 구조
4. 학습/추론 명령어
5. 결과 파일 열어보는 방법

---

## 1) 기본 설치 (Windows 기준)

## 1-1) Python + VS Code
- Python 3.10 또는 3.11 설치
- VS Code 설치
- VS Code 확장: Python (Microsoft)

## 1-2) 작업 폴더 만들기
예: `C:\vision-work`

## 1-3) 가상환경 생성/활성화
터미널(명령 프롬프트 또는 PowerShell):

```bash
cd C:\vision-work
python -m venv .venv
.venv\Scripts\activate
```

활성화되면 앞에 `(.venv)` 표시가 보입니다.

## 1-4) 패키지 설치
```bash
pip install --upgrade pip
pip install ultralytics opencv-python matplotlib numpy
```

## 1-5) 설치 확인
```bash
python -c "import ultralytics, cv2; print('ok')"
```

## 2) GPU 사용 확인 (있으면 좋음)

```bash
python -c "import torch; print(torch.cuda.is_available())"
```

- `True`: GPU 사용 가능
- `False`: CPU로 동작 (학습 느릴 수 있음)

※ 초반 실습은 CPU도 가능. 다만 학습 시간이 길어질 수 있음.

---

## 3) 디텍션(Detection) 실행 가이드

## 3-1) 데이터 구조

```text
detection_dataset/
  images/
    train/
    val/
  labels/
    train/
    val/
  data.yaml
```

- 이미지 1장마다 같은 이름의 라벨 txt 필요
- 예: `images/train/a001.jpg` ↔ `labels/train/a001.txt`

## 3-2) 라벨 형식 (YOLO)

```text
<class_id> <x_center> <y_center> <width> <height>
```

- 0~1 정규화 좌표
- 객체가 2개면 줄 2개

## 3-3) data.yaml 예시

```yaml
path: ./detection_dataset
train: images/train
val: images/val
names:
  0: scratch
  1: dent
```

## 3-4) 학습

```bash
yolo detect train data=detection_dataset/data.yaml model=yolov8n.pt epochs=50 imgsz=640
```

## 3-5) 추론

```bash
yolo detect predict model=runs/detect/train/weights/best.pt source=test_images conf=0.25
```

## 3-6) 결과 열어보기
- 결과 저장 경로: `runs/detect/predict/`
- 이미지 더블클릭해서 박스가 맞게 나왔는지 확인

---

## 4) 세그멘테이션(Segmentation) 실행 가이드

세그멘테이션은 "박스"가 아니라 "영역(마스크)"을 예측합니다.

## 4-1) 가장 쉬운 시작 방법
- Ultralytics의 세그 모델 사용: `yolov8n-seg.pt`
- 데이터 라벨은 세그 형식(폴리곤) 필요

## 4-2) 학습 명령어

```bash
yolo segment train data=seg_dataset/data.yaml model=yolov8n-seg.pt epochs=50 imgsz=640
```

## 4-3) 추론 명령어

```bash
yolo segment predict model=runs/segment/train/weights/best.pt source=test_images conf=0.25
```

## 4-4) 결과 열어보기
- 결과 저장 경로: `runs/segment/predict/`
- 불량 영역이 마스크로 제대로 칠해졌는지 확인

## 6) 라벨링 도구(중요)

디텍션/세그멘테이션은 라벨링이 핵심입니다.

추천 도구:
- **CVAT** (팀 협업 강함)
- **labelImg** (디텍션 박스 간단)
- **labelme** (세그 폴리곤 라벨링 쉬움)

초보 기준 추천:
- 디텍션 먼저(labelImg/CVAT)
- 그다음 세그멘테이션(labelme/CVAT)

## 7) 자주 나는 오류 + 해결

## 오류 A: `yolo` 명령이 안 먹음
- 가상환경 활성화 확인
- `pip install ultralytics` 다시 실행

## 오류 B: 라벨 관련 에러
- 이미지와 txt 파일명이 같은지 확인
- train/val 경로 오타 확인
- 빈 라벨 파일/깨진 파일 확인

## 오류 C: 학습은 되는데 성능이 너무 낮음
- 라벨 품질 문제 가능성 큼
- 클래스 불균형 확인
- 데이터가 실제 현장 분포와 다른지 확인

## 오류 D: 세그멘테이션 결과가 이상함
- 폴리곤 라벨 형식 점검
- 세그용 데이터셋인지 확인(디텍션 라벨과 형식 다름)

---

## 8) 디텍션 vs 세그멘테이션 선택 기준

디텍션 추천:
- 불량 위치 대략 알면 충분
- 빠른 개발/배포가 필요

세그멘테이션 추천:
- 불량 면적/형상을 정밀하게 봐야 함
- 경계 정확도가 중요한 공정

실무에서는 보통:
1) 디텍션으로 먼저 빠르게 도입
2) 필요 시 세그멘테이션으로 고도화
