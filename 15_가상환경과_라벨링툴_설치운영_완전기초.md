# 15. 가상환경과 라벨링 툴 설치/운영 (완전기초)

> 대상: "가상환경이 뭔지", "어디에 설치해야 하는지" 헷갈리는 상태의 입문자

---

## 1) 먼저 핵심 결론

- **labelImg / labelme**: 보통 **가상환경(venv) 안에 설치**해서 사용
- **YOLO(ultralytics)**: 보통 **가상환경 안에 설치**해서 사용
- **CVAT**: 보통 **Docker로 별도 실행** (가상환경과 성격이 다름)

즉,
- 파이썬 기반 로컬 툴 = venv
- 웹 서비스형 툴(CVAT) = Docker

---

## 2) 가상환경을 왜 쓰는가?

가상환경은 "프로젝트 전용 파이썬 공간"입니다.

장점:
1. 프로젝트마다 패키지 버전 분리
2. 충돌 방지 (A 프로젝트는 torch 2.2, B는 2.0 등)
3. 삭제/복구 쉬움

안 쓰면 생기는 문제:
- 다른 프로젝트가 갑자기 안 돌아감
- `pip install` 할 때 버전 충돌
- 원인 추적이 어려움

## 4) 가상환경 생성/활성화/종료 (Windows)

## 4-1) 생성
```bash
cd C:\vision-work\project-yolo-mask
python -m venv .venv
```

## 4-2) 활성화 (cmd/powershell)
```bash
.venv\Scripts\activate
```

성공하면 터미널 앞에 `(.venv)` 표시가 붙음.

## 4-3) 종료
```bash
deactivate
```

## 5) 라벨링 툴 설치 (venv 내부)

```bash
pip install --upgrade pip

```

확인:
```bash

```

## 6) 라벨링 툴 실행 방법

## 6-1) labelImg (디텍션 박스)
```bash
labelImg
```

- Open Dir: 이미지 폴더
- Change Save Dir: 라벨 폴더
- Format: YOLO
- 박스 그림 + 클래스 지정 + 저장

## 6-2) labelme (세그 폴리곤)
```bash
labelme
```

- Open: 이미지
- Polygon으로 경계 찍기
- 클래스명 지정 후 저장(JSON)

## 7) CVAT는 왜 다르게 설치하나?

CVAT는 단순 파이썬 툴이 아니라
**브라우저로 접속하는 서버형 라벨링 시스템**입니다.

그래서 보통:
- `pip install cvat` 방식이 아니라
- Docker Compose로 CVAT 서비스를 띄움

요약:
- labelImg/labelme = 로컬 앱 (venv)
- CVAT = 웹 서비스 (Docker)

---

## 8) YOLO 학습도 같은 venv에서 해도 되나?

네, 보통 **같은 프로젝트 venv**에서 해도 됩니다.

예시:
```bash
yolo detect train data=dataset/data.yaml model=yolov8n.pt epochs=50 imgsz=640
```

주의:
- 프로젝트가 완전히 달라지면 venv 분리 추천
- GPU/CUDA 버전 이슈가 있으면 프로젝트별 분리 필수

---

## 9) 자주 헷갈리는 포인트 정리

### Q1. 툴 설치할 때마다 venv 새로 만들어야 해?
- 같은 프로젝트면 기존 venv 사용
- 다른 프로젝트면 새 venv 권장

### Q2. 가상환경 안에서 설치했는데 명령어가 안 돼
- 활성화 안 된 상태일 수 있음
- `where labelImg` / `where python`으로 경로 확인

### Q3. 관리자 권한이 필요한가?
- 보통 필요 없음
- 회사 PC 정책으로 막히면 IT 정책 확인

### Q4. WSL, Windows 둘 다 있는데 어디서 할까?
- GUI 툴(labelImg/labelme)은 **Windows 터미널**이 편함
- 학습은 Windows/WSL 둘 다 가능하지만 한쪽으로 통일 추천

## 12) 실무 운영 패턴 (현업에서 실제로 이렇게 씀)

### 패턴 1) 개인 PoC / 과제형
- 툴: labelImg 또는 labelme
- 환경: 로컬 venv
- 흐름: 라벨링 → (필요 시 변환) → 코랩 학습
- 특징: 속도가 빠르고 세팅이 단순

### 패턴 2) 소규모 팀 파일공유형
- 툴: labelme + 공유 폴더(Drive/NAS/Git LFS)
- 환경: 각자 venv 통일
- 흐름: 라벨링 담당/학습 담당 분리
- 특징: 간단하지만 충돌관리(파일명, 버전) 규칙이 중요

### 패턴 3) 정식 실무 협업형
- 툴: CVAT
- 환경: Docker 서버 + 브라우저 접속
- 흐름: 작업자 라벨링 → 검수자 승인 → YOLO 포맷 export → 학습 파이프라인
- 특징: 품질관리/이력관리/역할분리가 쉬움

### 패턴 4) 빠른 실험 + 데이터 버전관리형
- 툴: Roboflow
- 환경: 웹 기반
- 흐름: 업로드 → 라벨링/증강 → 버전 생성 → YOLO export
- 특징: 빠른 반복 실험에 강함(보안정책 확인 필요)

## 13) 한 줄 요약

- **라벨링 툴(labelImg/labelme)과 YOLO는 가상환경 안에서 설치/실행**
- **CVAT는 Docker 기반 협업 툴로 별도 운영**
- **학습은 GPU가 없으면 코랩 활용**이 가장 현실적
